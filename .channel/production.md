# You Wouldn't Wanna Be — Production Standards

## Visual Style

**Classic American adult animation** (Family Guy seasons 10-20 aesthetic):
- Flat cel-shaded coloring with ZERO gradients
- Thick uniform black outlines on ALL elements (characters, objects, backgrounds)
- Simplified features, clean vector rendering
- No 3D rendering, no photorealistic elements, no shading variation, no lighting effects
- Static locked camera — no handheld, no shaky-cam, no drift
- Flat 2D compositions with painted layered backgrounds

## Clip Specs

- **Aspect ratio**: 9:16 vertical
- **Episode length**: 9–12 clips, **60–75 seconds** total (target 65s)
- **ALL audio generated natively by Veo 3.1** — narrator speech, SFX, ambience, music
- **No ElevenLabs** — no external TTS or music generation
- **Captions**: Remotion-rendered karaoke captions (Whisper transcription + transparent overlay)
- **Voice on frame 1** — narrator voiceover starts over the very first clip, no silent openings

| Clip Type | Duration | Resolution | Reference Images | Character | Audio |
|-----------|----------|------------|------------------|-----------|-------|
| HOOK | 4s | 720p | No (text-only) | Optional | Narrator + SFX + Music (Veo native) |
| SCENE | 4s or 8s | 1080p | Yes (angle-aware refs) | Yes | Narrator + SFX + Music (Veo native) |

- **Mix clip durations**: alternate 4s and 8s clips for visual rhythm
- **Typical episode**: 9–12 clips mixing 4s and 8s segments
- The skeleton is on camera in every SCENE clip but **never speaks**
- All narration is generated by Veo 3.1 as off-screen narrator voiceover

## Naming Rules

- NEVER reference copyrighted show names (Family Guy, Simpsons, etc.) in prompts — use "classic American adult animation style"
- Avoid "skeleton" in Veo prompts if it triggers safety filters
- Use **the figure** or **the translucent character** instead
- Every prompt with the character must include: "The on-screen character does NOT speak. Silent physical reactions only."
- The reference image handles visual identity
- All prompts must include the MANDATORY STYLE block as the first paragraph

## Clip Production Spec

Every clip in the storyboard must include these production fields:

- **Camera**: Static locked shot type (medium, close-up, wide, etc.) — no movement
- **Shot motivation**: WHY this framing — what it communicates emotionally
- **Composition**: Character blocking, rule of thirds, flat 2D staging
- **Depth layers**: foreground props, midground character space, background historical setting
- **Lighting + palette**: Flat even lighting. Named color palette per scene
- **Period texture**: At least one specific detail unique to this exact time and place
- **Background activity**: Other people doing things in the scene (flat 2D animated)
- **Visual transition**: How this clip connects to the previous clip
- **Character action**: What the skeleton is physically doing (cartoon reactions, gestures)
- **Dialogue direction**: Off-screen narrator's exact lines for this clip
- **Voice direction**: Narrator delivery notes (pace, tone, emotion)
- **Audio direction**: Music cues, SFX events, ambient sounds

## Models

- **Image generation**: NanoBanana Pro API (`nano-banana-pro` model, 2K resolution)
- **Video generation**: Veo 3.1 (`veo-3.1-generate-preview`)
- **Narrator voiceover**: Veo 3.1 native speech generation (off-screen British male narrator)
- **Background music**: Veo 3.1 native audio (directed via audio prompt)
- **Sound effects**: Veo 3.1 native audio (ambient/SFX generated with the video)
- **Transcription**: OpenAI Whisper (`whisper-1` — word-level timestamps for captions)
- **Caption rendering**: Remotion (`CaptionsOnly` composition — transparent ProRes 4444)
- **Safety settings**: `BLOCK_NONE` for all Gemini/Veo harm categories

## NanoBanana Pro Image Generation

- **API**: `POST https://nanobananapro.cloud/api/v1/image/nano-banana`
- **Poll results**: `POST https://nanobananapro.cloud/api/v1/image/nano-banana/result`
- **Model**: `nano-banana-pro` (20 credits per image at 1K, scales for 2K/4K)
- **Aspect ratio**: 9:16 for all frames
- **Image size**: 2K for scene keyframes, 1K for reference generation
- **Reference images**: up to 8 via multipart upload for character consistency
- **Mode**: `image-to-image` when using skeleton reference, `text-to-image` for establishing shots
- **All images must use the mandatory animation style block** — flat cel-shaded, thick outlines

## Veo 3.1 Duration/Resolution Constraints

- 4s clips: 720p only, supports first-frame conditioning from keyframe images
- 8s clips: 1080p, supports up to 3 reference images + `person_generation=allow_adult`
- Reference images and first/last frame modes **cannot be combined**
- All scene clips use 8s at 1080p with angle-aware character reference images

## Character Consistency

- The skeleton has the **same fixed appearance** in every episode (see `characters.md`)
- **Angle-aware reference images** at 4 angles, stored in `.channel/reference_images/`:
  - `skeleton_familyguy_front.png` — full-body front view (primary identity anchor)
  - `skeleton_familyguy_side.png` — full-body side profile
  - `skeleton_familyguy_threequarter.png` — full-body 3/4 view
  - `skeleton_familyguy_back.png` — full-body rear view
  - `skeleton_front_neutral.jpg` — original photorealistic source (fallback)
- Pipeline parses each video prompt for camera angle keywords and selects best 3 refs
- Passed as `VideoGenerationReferenceImage(reference_type="ASSET")`
- For 4s establishing shots: use first-frame conditioning from generated keyframe images
- Every prompt includes the full character visual consistency block

## Audio Pipeline

### Veo 3.1 Native Audio (ALL audio layers)

Veo 3.1 generates ALL audio natively from prompt direction:
- **Narrator voiceover**: Off-screen darkly amused British male via dialogue direction in prompts
- **Background music**: Dark cinematic underscore via music cues in audio direction
- **Sound effects**: Period-accurate environmental sounds via SFX direction
- **Ambience**: Continuous environmental atmosphere via ambience direction

There is NO external audio generation, NO audio mixing phase. The Veo clip contains all audio layers.

### LUFS Normalization

The final stitched video is normalized to **-14 LUFS** with **-1.0 dBTP** ceiling using ffmpeg's `loudnorm` filter. This ensures consistent loudness across episodes and platforms.

## Pipeline Phases

| Phase | Tool | What It Does |
|-------|------|-------------|
| `images` | NanoBanana Pro | Generate keyframe images for all clips (Family Guy style) |
| `videos` | Veo 3.1 | Generate video clips with native speech, SFX, ambience, music |
| `stitch` | ffmpeg | Concatenate clips + LUFS normalize to -14 LUFS |
| `captions` | Whisper + Remotion + ffmpeg | Transcribe audio, render karaoke captions, composite overlay |
| `publish` | Metricool | Multi-platform publish (Instagram, TikTok, YouTube) |
| `all` | All | Run full pipeline end to end |

### Captions Pipeline (Phase 4b)

1. **Whisper transcription**: Extract audio from stitched video, send to OpenAI Whisper API with `timestamp_granularities=["word"]` for word-level timing
2. **Remotion segment build**: Convert Whisper word timestamps to `HydratedSegment[]` JSON with karaoke-style HTML (current word highlighted, word-by-word progression)
3. **Remotion render**: Shell out to `npx remotion render CaptionsOnly` with `--codec=prores --prores-profile=4444` and `TRANSPARENT_EXPORT=true` to produce transparent ProRes 4444 MOV
4. **ffmpeg composite**: Overlay transparent caption MOV onto stitched video via `[0:v][1:v]overlay=0:0`

Requires `OPENAI_API_KEY` in `.env` and the Remotion project at `captions/captions-cloudflare/apps/remotion/` with dependencies installed.

## Image Prompt Structure

See `templates/image_prompt.txt`. Every image prompt must include:
1. MANDATORY STYLE block (first paragraph, word-for-word)
2. Format line with animation style
3. Camera composition (static, flat 2D framing)
4. Character visual consistency block (if character present)
5. Three depth layers (foreground / midground / background)
6. Flat lighting + named color palette
7. Period texture: one specific detail anchoring time and place
8. Background activity: people in scene (flat 2D animated)
9. Exclusion line (no text, no photorealistic, no 3D)

## Video Prompt Structure

See `templates/video_prompt.txt`. Every video prompt must include:
1. MANDATORY STYLE block (first paragraph, word-for-word)
2. Format + resolution line
3. Camera: static locked shot type, flat composition
4. Visual transition from previous clip
5. Character visual consistency block with physical action choreography
6. Flat lighting + named color palette
7. Three depth layers
8. Exclusion line
9. Dialogue direction (off-screen narrator's exact lines)
10. Voice direction (British male, delivery notes)
11. Audio direction: Music cues + SFX + Ambience
