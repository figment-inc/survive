## Learned User Preferences

- Scripts must tell one continuous story with causality and narrative thread — narration is generated as one continuous ElevenLabs audio file overlaid on the final stitched video; clip boundaries are visual cuts only, narration flows unbroken with mid-sentence splits
- Scripts must prioritize emotional storytelling over fact delivery — "story first, not encyclopedia"; each script must orbit ONE fatal detail (a door, a warehouse, a missing escort) threaded through every clip; a script that scatters across multiple unrelated facts per clip is a listicle, not a story; facts should be felt through the viewer's body, not read from a textbook; sensory texture (heat, dust, shaking, cold) and human moments (hope, decision, loss) are mandatory
- Scripts should feel like Kurzgesagt — a friendly expert explaining something incredible to a friend; hooks must immediately state the premise plainly (viewer thinks "oh no, that's a problem", not "wait, what?"); every sentence must build understanding of WHY the disaster happened (not atmosphere); endings give scale + cause + closure in one beat; the power comes from the viewer genuinely understanding how terrible it was, not from clever wordplay
- Endings must give real closure with scale + cause in one beat — not clever callbacks or wordplay; "Seven hundred buildings burned in twelve hours — because every horse was sick" is better than "The best fire department in America, beaten by a flu"
- Hooks must never be questions — always a single, plainly stated premise that the viewer immediately understands; cryptic fragments ("Twenty engines. Seven hundred men. Not one can move.") confuse rather than intrigue — the viewer should know what's happening, not solve a puzzle; the Constantinople hook is a good reference but not a mandatory template
- Visual consistency requires a global style reference image (`style_reference.png`) always passed as the first visual input to Gemini image generation — separate from and in addition to character angle references; in image-based (slideshow) mode, Veo is not used; in legacy Veo mode, non-character clips must NOT use ASSET mode (causes safety blocks), only first-frame conditioning; keyframe images must NOT occupy ASSET ref slots (displaces cross-clip style anchors)
- Second-person POV ("you") must never disappear in later clips — "you" must be the SUBJECT of at least one sentence in every catastrophe/conclusion clip; the disaster happens TO you, not around you while you watch statistics
- Use Claude Opus 4.6 (not Sonnet) for all content generation — always with streaming to avoid SDK timeouts; run Python commands with `-u` (unbuffered) for real-time progress output during long pipeline scripts
- The user works via plan files — implement them sequentially, do not edit the plan file itself, mark todos as in-progress, and complete all todos in a single session
- The user gives terse, lowercase instructions and expects immediate action without unnecessary clarifying questions — common one-shot commands include "push to main, run pipeline, publish to metricool" as a single workflow
- Script quality is iteratively critiqued — user expects a critique-then-fix cycle; prefer example-driven prompts (show 3 gold-standard scripts) over checklist-driven prompts (15 rules Claude optimizes against); rule-heavy prompts cause more problems than they solve — Claude ignores its own rules even when the BAD example is shown inline; structural prompt redesign (moving critical sections to the top, annotating examples) is more effective than adding more constraints
- Visual prompts must be narratively cohesive across images — each image must illustrate exactly what its narration sentence says; transcript is sacred and must never be modified; visuals mold to the transcript, not the other way around; cross-image sentence continuity requires same environment and camera angle family; include `NARRATION FOR THIS IMAGE` directive in every prompt

## Learned Workspace Facts

- Project is `survive.history` — an automated YouTube Shorts / TikTok / IG Reels pipeline for historical disaster episodes featuring an animated skeleton character
- Pipeline uses three-phase content generation: Writer-Critic-Rewriter pipeline (`writer.txt` → `critic.txt` → `rewriter.txt`) produces Kurzgesagt-style narration; critic scores 8 axes (Hook Clarity, Understanding, Narrative Thread, Closure, You-As-Subject, Narrator Restraint, Escalation Curve, Word Count); then `generate-prompts.txt` produces per-sentence image prompts with `NARRATION FOR THIS IMAGE` coupling; `--legacy-script` falls back to single-shot; `--script-file <path>` injects a pre-written script; intermediate artifacts (`00_draft_script.txt`, `00_critique.md`) saved to episode directory
- Pipeline: `n8n/run_pipeline.py` orchestrates topic selection (Claude via AWS Bedrock) → script generation (Claude Opus) → prompt generation (Claude Opus) → images (NanoBanana Pro / Gemini `gemini-3-pro-image-preview`, 1-2 per narration sentence) → audio (ElevenLabs TTS + music) → slideshow assembly (ffmpeg Ken Burns zoompan + xfade) → final mix (narration + music overlay) → captions (Remotion) → publish (Metricool); default mode is image-based slideshow; `--use-veo` flag falls back to legacy Veo 3.1 video pipeline; all Claude calls use AWS Bedrock (`AnthropicBedrock` client, model `anthropic.claude-opus-4-6-v1`, region `us-east-1`)
- Channel config files live in `.channel/`; default episode structure uses image-based slideshow mode: narration is split into sentences, 1-2 images per sentence assembled via Ken Burns zoompan effects with crossfade transitions; total ~38-40s video driven by narration length; 90-100 words target narration at ~2.6 wps; 3-beat narrative arc: Hook (clear premise) / Story (understanding) / Ending (scale + cause + closure); legacy Veo mode (5 clips x 8s = 40s raw) available via `--use-veo`
- ElevenLabs voice is "Dan" (British documentary narrator, voice ID `BHr135B5EUBtaWheVj8S`), actual TTS pacing ~2.6 words/second; image generation uses Google's NanoBanana Pro (`gemini-3-pro-image-preview` model) via the `google.genai` SDK; `lib/nanobanana.py` handles all image generation with style reference + character reference + style anchor chaining
- Image-based slideshow pipeline (`lib/slideshow.py`) parses narration into sentences, assigns 1-2 images per sentence (heuristic: 10+ words or action keywords = 2 images), syncs display durations to Whisper word-level timestamps, and assembles via ffmpeg `zoompan` (Ken Burns effects: slow zoom in/out, pan, combined) with `xfade` crossfade transitions; 7 cycling effects for visual rhythm; falls back to simple concat on xfade failure
- Style is flat 2D cel-shaded animation, 9:16 portrait for short-form platforms — photorealistic language (bokeh, film grain, depth of field) and the word "comedy" are banned from prompts and auto-sanitized; four character reference PNGs (front/side/threequarter/back) plus a global `style_reference.png` are passed to all generation calls, with the style ref always first; legacy Veo mode forced to 720p (1080p rejected), hard limit of 3 reference images per API call, real person names removed from prompts to avoid safety blocks
- Cross-clip transitions use ffmpeg xfade (0.3s default, 0.5s hook boundary, 1.5s dip-to-black at end) in `lib/mixer.py`; `mix_final_audio()` extends video by freezing the last frame when narration overflows instead of truncating audio mid-sentence; `validate_narration_timing()` applies atempo speedup (capped at 1.25x) as first resort; audio must never be cut off mid-sentence; in slideshow mode, `mix_final_audio()` detects silent input video and skips Veo audio mixing
- Karaoke captions pipeline (`lib/captions.py`) uses Remotion + Whisper: renders transparent ProRes overlay then composites via ffmpeg; caption position y=72% (lower-third), font auto-scaled to video width, max 3 words/segment; use original script text for captions instead of Whisper when available (Whisper loses words when audio overflows); visual QA (`lib/visual_qa.py`) is skipped in slideshow mode
- Metricool S3 direct upload is disabled on the current account (403 "Feature disabled") — pipeline falls back to GitHub Release download URLs; S3 video storage (`lib/s3.py`) syncs episode outputs to S3 after each pipeline phase (additive, local files remain); `S3_ENABLED` env var controls S3 sync
- Source citation pipeline (`n8n/system-prompts/sources.txt`) generates 3-6 fact/source pairs after script finalization, saved as `05_sources.json`; formatted as first comment on social posts via Metricool's `firstCommentText` field; comments use concise format (title + hyperlink per line, no verbose fact quotes); `publish.py` auto-loads sources from episode directory when `--first-comment` is not provided
