# AGENTS.md

## Learned User Preferences

- Scripts must tell one continuous story with causality and narrative thread — narration is generated as one continuous ElevenLabs audio file overlaid on the final stitched video; clip boundaries are visual cuts only, narration flows unbroken with mid-sentence splits
- Scripts must prioritize emotional storytelling over fact delivery — "story first, not encyclopedia"; each script must orbit ONE fatal detail (a door, a warehouse, a missing escort) threaded through every clip; a script that scatters across multiple unrelated facts per clip is a listicle, not a story; facts should be felt through the viewer's body, not read from a textbook; sensory texture (heat, dust, shaking, cold) and human moments (hope, decision, loss) are mandatory
- Scripts should feel like PBS documentaries — open with the routine/competence (not the event), introduce the turning point as mundane furniture (not drama), let the viewer discover the irony; the power comes from restraint and the narrator trusting the viewer to feel it without being told to; the fatal detail must read like an accountant's inventory — no foreshadowing, no "until it doesn't", no narrator commentary
- Endings must land on a definitive beat that echoes the hook — weak, unresolved endings are not acceptable
- Hooks must never be questions — always a single, calmly stated line that creates a "wait, what?" reaction and is engaging for TikTok/Shorts; hooks must be genuinely interesting and scroll-stopping but do NOT need to be "impossibilities" or "paradoxes" — rigid formula rules cause Claude to optimize for the wrong thing; the Constantinople hook ("A thousand years of walls. One unlocked door.") is a good reference, not a mandatory template
- Visual consistency requires a global style reference image (`style_reference.png`) always passed as the first visual input to both Gemini image and Veo video generation — separate from and in addition to character angle references; non-character clips must NOT use ASSET mode (causes safety blocks), only first-frame conditioning; keyframe images must NOT occupy ASSET ref slots (displaces cross-clip style anchors)
- Second-person POV ("you") must never disappear in later clips — "you" must be the SUBJECT of at least one sentence in every catastrophe/conclusion clip; the disaster happens TO you, not around you while you watch statistics
- Use Claude Opus 4.6 (not Sonnet) for all content generation — always with streaming to avoid SDK timeouts; run Python commands with `-u` (unbuffered) for real-time progress output during long pipeline scripts
- The user works via plan files — implement them sequentially, do not edit the plan file itself, mark todos as in-progress, and complete all todos in a single session
- The user gives terse, lowercase instructions and expects immediate action without unnecessary clarifying questions — common one-shot commands include "push to main, run pipeline, publish to metricool" as a single workflow
- Script quality is iteratively critiqued — user expects a critique-then-fix cycle; prefer example-driven prompts (show 3 gold-standard scripts) over checklist-driven prompts (15 rules Claude optimizes against); rule-heavy prompts cause more problems than they solve — Claude ignores its own rules even when the BAD example is shown inline; structural prompt redesign (moving critical sections to the top, annotating examples) is more effective than adding more constraints
- Visual prompts must be narratively cohesive across clips — each clip's image/video prompt should reference shared visual elements (recurring props, environment continuity, character positioning) rather than generating isolated pretty images; deliberate visual storytelling, not random composition

## Learned Workspace Facts

- Project is `survive.history` — an automated YouTube Shorts / TikTok / IG Reels pipeline for historical disaster episodes featuring an animated skeleton character
- Pipeline uses three-phase content generation: Writer-Critic-Rewriter pipeline (`writer.txt` → `critic.txt` → `rewriter.txt`) produces scored/critiqued narration as the default; critic scores 10 axes (Hook Impact, Sensory Immersion, Narrative Thread, Context Clarity, You-As-Subject, Narrator Restraint, Escalation Curve, Twist Payoff Power, Rhythm Shift, Word Count); the zero-jump rule and context-bridging are enforced across all three phases; then `generate-prompts.txt` takes the approved script and produces image/video prompts + full episode JSON; `--legacy-script` falls back to single-shot `generate-script.txt`; `--legacy-prompt` falls back to the old single-pass `generate-episode-legacy.txt`; `--script-file <path>` injects a pre-written script and skips generation entirely; intermediate artifacts (`00_draft_script.txt`, `00_critique.md`) are saved to the episode directory
- Pipeline: `n8n/run_pipeline.py` orchestrates topic selection (Claude) → script generation (Claude Opus) → prompt generation (Claude Opus) → images (Gemini `gemini-3-pro-image-preview`) → audio (ElevenLabs TTS + music) → video (Veo 3.1) → mix/stitch (ffmpeg) → captions (Remotion) → publish (Metricool); phases run strictly sequentially with no parallelism
- Channel config files live in `.channel/`: `channel.md` (series bible), `production.md` (specs), `cinematography.md` (visual guide), `templates/` (prompt templates); episode structure: 5 clips (all 7s = 35s raw, ~33s effective after crossfades), 75-85 words target narration at 2.4 wps; 3-beat narrative arc: Hook (0-5s) / Easy Explanation (5-28s) / Twist Payoff (28-35s)
- ElevenLabs voice is "Dan" (British documentary narrator, voice ID `BHr135B5EUBtaWheVj8S`), measured pacing ~2.4 words/second; image generation uses Google's `gemini-3-pro-image-preview` model via the `google.genai` SDK (`GEMINI_API_KEY` from `.env`)
- Veo generation is forced to 720p for all clips — 1080p is rejected by the extension API and causes `INVALID_ARGUMENT` errors with reference images; hard limit of 3 reference images per API call; all clips are normalized to 720x1280 before stitching; real person names (emperors, historical figures) must be removed from video prompts to avoid "celebrity reference" safety blocks — use generic descriptions like "the commander" or "the lone figure"
- Style is flat 2D cel-shaded animation, 9:16 portrait for short-form platforms — photorealistic language (bokeh, film grain, depth of field) and the word "comedy" are banned from prompts and auto-sanitized in `lib/veo.py`; four character reference PNGs (front/side/threequarter/back) plus a global `style_reference.png` (converted from AVIF) are passed to all generation calls, with the style ref always first
- Cross-clip transitions use ffmpeg xfade (0.3s default, 0.5s hook boundary, 1.5s dip-to-black at end) in `lib/mixer.py`; transitions consume ~1.7s total for 5 clips (0.5 + 3*0.3 + dip-to-black), and `validate_narration_timing()` applies atempo speedup (capped at 1.25x) when narration overflows the effective video window; the word-count trim loop has 2 retries which may be insufficient for large overages
- Karaoke captions pipeline (`lib/captions.py`) uses Remotion + Whisper: renders transparent ProRes overlay then composites via ffmpeg; caption position y=72% (lower-third), font auto-scaled to video width, max 3 words/segment; wired as Phase 4b after stitching
- Visual QA pipeline (`lib/visual_qa.py`) extracts multi-point frames from each clip and scores drift severity; QA auto-regen is on by default (`--no-qa-regen` to disable); a post-QA quality gate blocks publish if >2 clips are at severity >= 4 (`--force-publish` to override)
- Metricool S3 direct upload is disabled on the current account (403 "Feature disabled") — pipeline automatically falls back to GitHub Release download URLs as media source for publishing
- Pipeline has no standalone publish-only mode — re-running with all phases skipped regenerates the slug and can't find existing video files; publishing an already-generated episode requires calling `create_github_release()` and `publish_to_metricool_with_upload()` directly via a custom script
